## Product Requirements Document: 하루 만원 살기 플래너

**Author**: User

### 1. Executive Summary

"하루 만원 살기 플래너"는 10~20대를 타겟으로 한 심플 미니멀 UI 기반의 일일 예산 관리 앱이다. 기존 가계부 앱과 달리 "하루 1만원"이라는 명확한 목표를 제시하고, 숫자 자체가 감정을 표현하는 독창적인 UX로 절약 습관을 형성할 수 있도록 설계한다. Flutter 기반으로 iOS/Android 동시 지원하며, 2주 내 MVP를 출시한다.

---

### 2. Background & Context

**문제 공간**:

- 10~20대(학생, 사회초년생)는 소득이 제한적이지만 소비 유혹이 많음
- 기존 가계부 앱은 "기록" 중심으로 지루하고, 절약 동기부여가 약함
- "짠테크", "만원 살기" 등 절약 챌린지가 SNS에서 유행하지만, 이를 체계적으로 지원하는 앱이 없음

**시장 맥락**:

- 기존 가계부 앱(뱅크샐러드, 편한가계부 등)은 30~40대 중심, 종합 자산관리에 초점
- MZ세대 타겟 절약 앱 시장은 아직 블루오션
- "미니멀 디자인 + 감정적 숫자 UX"는 복잡한 앱에 피로감을 느끼는 10~20대에게 차별화된 어필 포인트

**디스커버리 결과 요약**:

- 디스커버리 플랜(docs/discovery_plan.md)에서 9개 기능 아이디어 도출
- 핵심 검증 가정: 수요 존재(V2), 시장 규모(B1), 바이럴 가능성(G1)
- MVP는 검증과 동시에 빠르게 출시하는 전략

### 3. Objectives & Success Metrics

**Goals**:

1. 2주 내 MVP 출시 — 핵심 기능 3개로 앱스토어 배포 가능한 상태
2. 10~20대가 "매일 열고 싶은" 절약 앱 경험 제공
3. 숫자가 감정을 표현하는 미니멀 UI로 기존 가계부 앱과 차별화

**Non-Goals (MVP 범위 밖)**:

1. SMS/알림 자동 파싱 — 기술적 복잡도 높음, 2차 릴리즈에서 검토
2. 카테고리별 소비 패턴 분석 — 데이터 축적 후 추가
3. 감정 기반 지출 태그 — UX 검증 후 추가
4. 소셜 기능 (친구 챌린지, 랭킹) — MVP 이후
5. 수익화 (광고 삽입) — 사용자 확보 후 적용
6. 사용자 맞춤 금액 설정 — 기본값 1만원 고정, 추후 변경 기능 추가

**Success Metrics**:

| 지표                   | 현재 | 목표                            | 측정 방법          |
| ---------------------- | ---- | ------------------------------- | ------------------ |
| 앱 설치 수             | 0    | 출시 1개월 내 1,000건           | 앱스토어 콘솔      |
| DAU (일일 활성 사용자) | 0    | 설치 대비 30%                   | Firebase Analytics |
| 일일 지출 기록률       | 0    | DAU 중 70% 이상이 1건 이상 기록 | 앱 내 이벤트       |

### 5. User Stories & Requirements

**P0 — Must Have (MVP 필수)**:

| #    | 사용자 스토리                                                         | 수용 기준                                                                                                                                                                                                    |
| ---- | --------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| P0-1 | 사용자로서, 앱을 열면 오늘 남은 예산(1만원 기준)을 즉시 확인하고 싶다 | - 앱 실행 시 메인 화면에 남은 금액이 큰 숫자로 중앙에 표시됨 - 남은 금액 상태에 따라 숫자 크기/색상이 변화함 (여유: 크고 안정된 톤, 빠듯: 작아지며 미세 떨림, 초과: 붉은 톤) - 하루 기준은 자정(00:00)에 리셋됨 |
| P0-2 | 사용자로서, 지출을 최대한 빠르게 기록하고 싶다                        | - 메인 화면에서 1탭으로 지출 입력 화면 진입 - 금액 입력 → 카테고리 선택(식비/교통/카페/쇼핑/기타) → 저장까지 3탭 이내 - 카테고리는 이모지 아이콘+텍스트로 직관적 선택 - 저장 시 메인 화면의 남은 금액이 즉시 차감됨 |
| P0-3 | 사용자로서, 오늘의 지출 내역을 확인하고 싶다                          | - 메인 화면 하단에 오늘 지출 리스트 표시 - 각 항목: 금액, 카테고리, 시간 표시 - 항목 스와이프로 삭제 가능 - 항목 탭으로 수정 가능                                                                            |
| P0-4 | 사용자로서, 지출 기록 시 기분 좋은 피드백을 받고 싶다                 | - 지출 저장 시 금액 차감 카운트다운 애니메이션 - 바운스 효과 및 파티클 등 마이크로 인터랙션 - 시간대별 배경 톤 변화: 아침(크림)/점심(화이트)/저녁(웜그레이)/초과(핑크틴트) - 라이트/다크 모드 모두 지원    |
| P0-5 | 사용자로서, 앱 데이터가 기기에 안전하게 저장되길 원한다               | - 모든 데이터는 로컬(기기 내)에 저장 - 서버 통신 없음 (오프라인 퍼스트) - 앱 재시작 시 데이터 유지                                                                                                           |
| P0-6 | 사용자로서, 홈 화면 위젯에서 남은 금액을 바로 확인하고 기록하고 싶다  | - iOS/Android 홈 화면 위젯 지원 - 위젯에 오늘 남은 금액 표시 - 위젯에서 바로 지출 기록 진입 가능 - 위젯 탭 시 앱 실행                                                                                       |

| #    | 사용자 스토리                                               | 수용 기준                                                                                                                                              |
| ---- | ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| P1-1 | 사용자로서, 오늘 아낀 돈을 내일로 이월하고 싶다             | - 오늘 8,000원 사용 시 → 내일 예산 12,000원으로 자동 반영 - 오늘 12,000원 사용(초과) 시 → 내일 예산 8,000원으로 자동 차감 - 이월 내역을 확인할 수 있음 |
| P1-2 | 사용자로서, 밤에 오늘의 절약 결과를 알림으로 받고 싶다      | - 매일 밤 9시(기본) 로컬 푸시 알림 - 오늘 남은 금액 메시지 (예: "오늘 3,200원 남겼어요!") - 알림 시간 설정 가능 - 알림 on/off 토글                    |

| P1-4 | 사용자로서, 절약 결과를 SNS에 공유하고 싶다                 | - 카카오톡/인스타 공유 카드 자동 생성 - 오늘 남은 금액, 연속 성공 일수 포함 - 미니멀 디자인 공유 카드                                                  |
| P1-5 | 사용자로서, 만원 이내로 성공하면 도토리를 모으고 싶다       | - 하루 만원 이내 성공 시 도토리 1개 획득 - 아낀 금액에 따라 보너스 도토리 지급 (5,000원 이상 절약 시 +1) - 메인 화면에 보유 도토리 수 표시 - 도토리 획득 시 마이크로 인터랙션 애니메이션 |

### 6. Solution Overview

**기술 스택**:

- **프레임워크**: Flutter (iOS/Android 동시 지원)
- **상태관리**: Riverpod
- **로컬 저장소**: Hive 또는 Isar (NoSQL, 빠른 읽기/쓰기)
- **라우팅**: GoRouter
- **알림**: flutter_local_notifications
- **홈 위젯**: home_widget (iOS WidgetKit / Android AppWidget)
- **아키텍처**: Clean Architecture (data/domain/presentation)

**핵심 설계 결정**:

| 결정             | 선택                                | 이유                                        |
| ---------------- | ----------------------------------- | ------------------------------------------- |
| 서버 vs 로컬     | 로컬 전용                           | MVP 속도, 개인정보 우려 제거, 서버 비용 0   |
| 일일 기준 시간   | 자정 (00:00)                        | 직관적, 사용자 혼란 최소화                  |
| 기본 예산        | 10,000원 고정                       | 앱 정체성, 추후 커스텀 추가                 |
| 숫자 감정 표현   | 3단계 (여유/빠듯/초과)              | 캐릭터 없이 숫자 자체로 상태 전달           |
| 카테고리         | 5개 고정 이모지 아이콘 (식비/교통/카페/쇼핑/기타) | 빠른 선택, 분석 피로 감소      |
| 테마             | 라이트/다크 모드 모두 지원          | 사용자 환경 다양성 대응                     |
| 배경 톤          | 시간대별 4단계 자동 변화            | 미니멀 디자인 내 시각적 감성 제공           |

**숫자 감정 표현 스펙**:

| 상태     | 조건                   | 숫자 크기    | 색상           | 애니메이션           |
| -------- | ---------------------- | ------------ | -------------- | -------------------- |
| 여유     | 남은 금액 5,000원 이상 | 크고 굵게    | 안정적인 다크  | 없음 (정적)          |
| 빠듯     | 남은 금액 1~4,999원    | 약간 작게    | 중간 톤        | 미세 떨림(shake)     |
| 초과     | 남은 금액 0원 미만     | 작게         | 붉은 톤        | 떨림 + 붉은 강조     |

**시간대별 배경 톤**:

| 시간대       | 배경 톤     |
| ------------ | ----------- |
| 06:00~11:59  | 크림(아침)  |
| 12:00~17:59  | 화이트(점심) |
| 18:00~23:59  | 웜그레이(저녁) |
| 초과 상태    | 핑크틴트    |

**화면 구성 (MVP)**:

```
[메인 화면]
├── 남은 금액 숫자 (중앙, 상태에 따라 크기/색상/애니메이션 변화)
├── 보유 도토리 수 & 스트릭 일수 표시
├── 지출 추가 버튼 (FAB)
├── 오늘 지출 리스트
└── 하단 네비게이션

[지출 입력 화면]
├── 금액 입력 (숫자 키패드)
├── 카테고리 선택 (5개 이모지 아이콘)
└── 저장 버튼 (저장 시 차감 카운트다운 + 마이크로 인터랙션)

[캘린더/기록 화면]
├── 월간 캘린더 (날짜별 성공/실패 + 스트릭 연속선 표시)
├── 날짜 선택 시 해당일 지출 내역
└── 현재 칭호 표시

[업적 화면]
├── 배지 컬렉션 (달성/미달성 실루엣)
├── 칭호 목록 & 현재 칭호
└── 도토리 획득 히스토리

[설정 화면]
├── 알림 설정 (시간, on/off)
├── 테마 설정 (라이트/다크)
├── 데이터 초기화
└── 앱 정보
```
