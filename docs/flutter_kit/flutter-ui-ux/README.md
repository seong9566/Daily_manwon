# flutter-ui-ux

이 스킬은 Flutter 화면 UI/UX를 설계하거나 다듬을 때, 일관된 디자인 토큰과 재사용 가능한 위젯 구조를 유지하기 위한 기준입니다.

## 언제 사용하는가

- 새 화면이나 컴포넌트를 설계할 때
- 기존 UI를 정리하거나 공통화할 때
- 반응형 레이아웃, 접근성, 상태 UI를 개선할 때
- 디자인 토큰 사용 여부를 점검할 때

## 핵심 요약

- 하드코딩 대신 프로젝트의 색상, 간격, 타이포그래피 토큰을 사용합니다.
- 복잡한 UI 조각은 `_build*()` 함수보다 별도 위젯 클래스로 분리합니다.
- 모바일 우선으로 설계하고 태블릿/데스크톱 대응을 고려합니다.
- 로딩, 에러, 빈 상태와 접근성까지 포함해 화면 품질을 완성합니다.
