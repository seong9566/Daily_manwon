# git-commit

이 스킬은 Conventional Commits 규칙에 따라 변경 사항을 의미 단위로 나누어 깔끔하게 커밋할 때 사용합니다.

## 언제 사용하는가

- 변경 파일을 스코프별로 나눠 여러 커밋으로 정리할 때
- 커밋 메시지를 일관된 형식으로 작성할 때
- 깔끔한 Git 히스토리를 남기고 싶을 때

## 핵심 요약

- 한 커밋에는 한 스코프만 담습니다.
- 커밋 제목 형식은 `<type>(<scope>): <message>`를 사용합니다.
- `type`과 `scope`는 영어, `message`와 본문은 한글로 작성합니다.
- 본문은 `What`, `Why`, `How`를 짧은 명사형 표현으로 정리합니다.
