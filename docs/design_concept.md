### 1. 캐릭터 컨셉

**마스코트**: 다람쥐 캐릭터
**컨셉 키워드**: 도토리를 모으는 다람쥐 = 돈을 아끼는 사용자

**캐릭터 이름 후보**:
- 도토리 (도토리를 모으는 다람쥐)
- 만이 (만원의 "만" + 귀여운 어미)
- 또리 (도토리에서 파생)
- 다솜 (다람쥐 + 사랑)

**캐릭터 상태 (3단계)**:

| 상태 | 조건 | 표정/포즈 | 메시지 예시 |
|------|------|----------|------------|
| 행복 | 남은 금액 ≥ 5,000원 | 볼 빵빵, 도토리를 안고 있음, 반짝이는 눈 | "오늘도 알뜰살뜰!" |
| 걱정 | 남은 금액 1,000~4,999원 | 한쪽 눈 찡긋, 도토리를 꽉 쥐고 있음, 땀방울 | "조금만 아껴보자..!" |
| 울상 | 남은 금액 < 1,000원 또는 초과 | 눈물 글썽, 빈 도토리 주머니, 납작 엎드림 | "으앙... 도토리가 없어..." |

**추가 리액션 (지출 기록 시)**:
- 소액 지출 (≤ 2,000원): 고개 끄덕, "괜찮아!"
- 중간 지출 (2,001~5,000원): 살짝 놀란 표정, "음...!"
- 대형 지출 (> 5,000원): 놀라서 뒤로 넘어가는 포즈, "헉!"

### 3. 타이포그래피

**폰트 선택**:
- 한글: Pretendard (깔끔, 가독성 좋음, 무료)
- 숫자/금액: Pretendard Bold 또는 Poppins Bold (임팩트)
- 대안: NanumSquareRound (둥글둥글 귀여운 느낌)

**폰트 크기 체계**:

| 용도 | 크기 | 굵기 | 예시 |
|------|------|------|------|
| 남은 금액 (메인) | 40sp | Bold | "7,200원" |
| 섹션 타이틀 | 18sp | SemiBold | "오늘의 지출" |
| 리스트 항목 금액 | 16sp | Medium | "-3,500원" |
| 리스트 항목 설명 | 14sp | Regular | "점심 김밥" |
| 캡션/시간 | 12sp | Regular | "오후 2:30" |
| 캐릭터 말풍선 | 14sp | Medium | "오늘도 알뜰살뜰!" |

### 4. UI 컴포넌트 스타일

**전체 무드**: 깔끔 미니멀 + 귀여운 포인트

**카드**:
- 모서리 반경: 16px (부드러운 라운드)
- 그림자: 매우 연하게 (elevation 1~2)
- 패딩: 16px

**버튼**:
- FAB (지출 추가): 둥근 원형, 프라이머리 오렌지, 도토리 또는 + 아이콘
- 카테고리 버튼: 둥근 사각형 칩, 선택 시 해당 카테고리 색상으로 채워짐
- 저장 버튼: 풀 와이드, 프라이머리 오렌지, 라운드

**아이콘**:
- 스타일: 라운드 라인 아이콘 (Material Rounded 또는 커스텀)
- 카테고리 아이콘:
  - 식비: 🍚 밥그릇
  - 교통: 🚌 버스
  - 카페: ☕ 커피잔
  - 쇼핑: 🛍️ 쇼핑백
  - 기타: 📦 상자

**애니메이션**:
- 페이지 전환: 슬라이드 (300ms)
- 금액 차감: 숫자 카운트다운 애니메이션 (500ms)
- 캐릭터 리액션: 바운스/스프링 (400ms)
- 리스트 아이템 등장: 페이드 + 슬라이드업 (200ms)

### 5. 화면별 디자인 방향

#### 메인 화면

```
┌─────────────────────────┐
│                         │
│    ┌───────────────┐    │
│    │   🐿️ 다람쥐    │    │
│    │  (캐릭터 영역)  │    │
│    │               │    │
│    │  "오늘도       │    │
│    │   알뜰살뜰!"   │    │
│    └───────────────┘    │
│                         │
│     남은 금액            │
│     ₩ 7,200             │
│     ━━━━━━━━░░ 72%      │
│                         │
│  ── 오늘의 지출 ────────  │
│                         │
│  🍚 점심 김밥    -3,500원 │
│     오후 12:30           │
│                         │
│  ☕ 아이스 아메  -1,300원  │
│     오후 3:15            │
│                         │
│              ┌───┐      │
│              │ + │ FAB  │
│              └───┘      │
│                         │
│  🏠    📅    ⚙️          │
│  홈   캘린더  설정       │
└─────────────────────────┘
```

- 상단 50%: 캐릭터 + 남은 금액 (앱의 핵심)
- 프로그레스 바: 남은 비율 시각화 (파스텔 그린 → 옐로 → 레드)
- 하단 50%: 오늘 지출 리스트 (스크롤 가능)
- FAB: 오른쪽 하단, 프라이머리 오렌지

#### 지출 입력 화면 (바텀시트)

```
┌─────────────────────────┐
│  지출 기록하기      ✕    │
│                         │
│  ₩ 3,500                │
│  ─────────────────────  │
│                         │
│  🍚    🚌    ☕    🛍️   📦  │
│  식비  교통  카페  쇼핑  기타│
│                         │
│  ┌─────────────────────┐│
│  │    저장하기          ││
│  └─────────────────────┘│
│                         │
│  [1] [2] [3] [⌫]       │
│  [4] [5] [6] [00]      │
│  [7] [8] [9] [0]       │
└─────────────────────────┘
```

- 메인 화면 위에 바텀시트로 올라옴 (전체 페이지 전환 X)
- 숫자 키패드는 커스텀 (네이티브 키보드 대신)
- 카테고리 선택 → 색상 변경으로 피드백

### 7. 디자인 원칙

1. **3초 룰**: 앱을 열고 3초 안에 남은 금액을 파악할 수 있어야 한다
2. **원탭 기록**: 지출 기록까지 최소 탭으로 도달
3. **캐릭터가 주인공**: UI는 미니멀하게, 캐릭터가 감정적 연결의 중심
4. **색상 = 정보**: 프로그레스 바, 캘린더 dot 등 색상으로 상태를 직관적으로 전달
5. **여백의 미**: 요소 간 충분한 여백으로 깔끔한 인상 유지
